# BOM Checklist Manager

功能 BOM 与测试清单管理器。项目用于展示对 BOM 清单、需求拆解、流程标准化和测试验收的理解。

## 项目定位

BOM 不只可以表示硬件物料，也可以抽象为“功能清单”。本项目以校园小程序为例，将业务功能拆成：

- 所属模块
- 子功能
- 页面 / 入口
- 数据字段
- 依赖关系
- 测试点
- 交付状态

## 核心功能

- 新增功能 BOM 条目
- 示例数据快速载入
- 完成率、待确认数量等状态统计
- 删除和清空清单
- 自动生成项目 SOP
- 导出 Markdown 项目文档

## 本地运行

直接打开 `index.html` 即可运行。

```text
bom-checklist-manager/
├── index.html
├── assets/
│   ├── app.js
│   └── styles.css
└── README.md
```

## 简历写法

**功能 BOM 与测试清单管理器｜个人项目**

设计并实现功能 BOM 管理工具，基于校园小程序项目将闲置交易、拼车、失物招领和消息中心等功能拆解为模块、页面、字段、依赖关系和测试点；通过状态统计和 Markdown 导出，实现需求清单整理、功能核对、测试验收和项目文档沉淀，体现结构化拆解、BOM 整理和 SOP 标准化能力。

## 与岗位匹配点

- 对应“备件清单整理”：将信息统一整理成结构化清单
- 对应“需求分析与流程拆解”：按模块、字段、依赖和状态拆解功能
- 对应“SOP 与文档沉淀”：自动生成可复用流程和项目文档
- 对应“AI Agent 思维”：为后续接入大模型生成需求说明、测试清单和 Prompt 留出扩展空间
